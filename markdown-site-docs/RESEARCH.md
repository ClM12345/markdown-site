# Markdown 网站需求研究

## 1. 项目目标
做一个集 Markdown 介绍、语法学习、在线编辑、实时预览、文件导入导出于一体的网站。

## 2. 这个项目要解决什么问题
很多人知道 Markdown，但不会系统使用。
现有工具往往有两个问题：
1. 纯文档站只能看教程，不能直接练习
2. 纯编辑器只适合会用的人，新手门槛高

所以本项目希望解决：
- 让新手快速理解 Markdown 是什么、怎么用
- 让用户可以直接上传和读取 `.md` 文件
- 让用户可以在线编辑 Markdown
- 让用户实时看到渲染效果
- 让用户方便导出处理后的内容

## 3. 目标用户
- 想入门 Markdown 的新手
- 需要经常写 Markdown 的学生、程序员、写作者
- 需要快速查看 `.md` 文件内容的用户
- 想找一个轻量在线 Markdown 工具的人

## 4. 典型使用场景
### 场景 1：学习 Markdown
用户进入网站，阅读 Markdown 的介绍和语法示例，并在编辑区实时练习。

### 场景 2：查看 Markdown 文件
用户拖入一个 `.md` 文件，网站自动解析并渲染出排版后的内容。

### 场景 3：编辑和导出
用户修改 Markdown 内容，然后导出为新的 `.md` 文件、HTML 文件，或者打印为 PDF。

### 场景 4：写作辅助
用户使用模板、快捷插入语法块、目录生成等功能，更高效地写 Markdown。

## 5. 调研结论
理想的 Markdown 网站应该同时具备：
- 学习功能
- 编辑功能
- 实时预览功能
- 文件处理功能
- 导出能力

## 6. 差异化方向
本项目的差异化不在“功能最复杂”，而在“上手简单、学习和工具合一”：
- 新手能学
- 老手能用
- 页面清晰
- 操作直观
- 文件处理顺手

## 7. 核心需求
1. Markdown 基础介绍
2. Markdown 语法教学与示例
3. 在线编辑器
4. 实时预览
5. 导入 `.md` 文件
6. 导出 `.md` 文件
7. 导出 HTML / PDF
8. 自动保存内容

## 8. 第一版不做什么
- 不做多人协作
- 不做账号系统
- 不做云端同步
- 不做复杂权限管理
- 不做大型知识库
- 不做 AI 自动生成文章（后续可扩展）
