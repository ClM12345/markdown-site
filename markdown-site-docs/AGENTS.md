# Markdown 网站 AI 开发指令

## 项目概述
这是一个 Markdown 学习与编辑网站。
网站包含 Markdown 的介绍和使用方法，同时支持读取、编辑、预览和导出 Markdown 文件。

第一版目标：
1. Markdown 基础介绍
2. Markdown 语法教学
3. 在线编辑器
4. 实时预览
5. 导入 `.md` 文件
6. 导出 `.md`
7. 导出 HTML / PDF
8. 本地自动保存

## 技术栈
- React
- TypeScript
- Vite
- Tailwind CSS
- react-markdown
- remark-gfm

## 开发规范
- 使用 TypeScript，保证核心类型清晰
- 组件使用函数式组件 + Hooks
- 优先组件化拆分，避免巨型组件
- 保持 UI 简洁现代
- 第一版以前端本地能力为主，不接后端
- 文件处理尽量在浏览器端完成

## 代码风格
- 组件名使用 PascalCase
- 变量和函数使用 camelCase
- 常量使用 UPPER_SNAKE_CASE
- 工具函数和 UI 逻辑尽量分离
- 避免重复代码
- 避免过度复杂抽象

## 页面要求
- 首页展示产品定位和主要功能
- 介绍页说明 Markdown 是什么
- 教学页展示常见语法和示例
- 编辑器页提供输入、预览、导入导出等能力
- 页面支持桌面端和移动端适配

## 编辑器要求
- 左侧编辑，右侧预览（移动端可上下布局）
- 支持默认示例内容
- 支持清空内容
- 支持恢复示例
- 支持自动保存到 LocalStorage

## 文件功能要求
- 支持读取 `.md` 文件
- 支持拖拽上传
- 支持导出 `.md`
- 支持导出 HTML
- 支持打印 / 导出 PDF
- 错误文件要有提示

## 测试要求
每完成一个功能后检查：
1. 页面是否正常运行
2. Markdown 是否正确渲染
3. 文件导入是否成功
4. 导出文件是否可用
5. 自动保存是否生效
6. 移动端布局是否可用
7. 深色模式是否正常（如果已实现）

## 注意事项
- 优先完成 MVP，不要过早扩展复杂功能
- 每次只修改当前任务相关文件
- 保持现有功能不被破坏
- 如果要重构，先说明原因
- 输出代码时优先保证可运行

## AI 协作要求
- 如果发现布局重复，请主动抽成组件
- 如果问题反复出现，请先分析原因，再给出最小修改方案
- 不要一次性修改过多无关文件
- 每完成一个阶段后，列出修改的文件和实现的功能
